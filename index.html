Criação de um Banco de Dado

CREATE DATABASE nome_do_bancodedados
Esse comando é usado para criar um banco de dados.

Criação de uma Tabela no Banco de Dados:

CREATE TABLE pessoas ( nome VARCHAR(30), idade TINYINT(3), sexo CHAR(1), peso FLOAT, altura FLOAT, nacionalidade VARCHAR(20) ) default charset = utf8mb3
Criando uma tabela chamada pessoas, passando as colunas que eu quero ter nela e passando o utf8 como padrão default da tabela, o utf8 define como o se armazena caracteres de texto, e é importante para garantir a compatibilidade e o uso adequado de caracteres em sua aplicação (não é obrigatorio definir no final o utf8 para criar uma tabela).

Apagar um Banco de Dados:

DROP DATABASE cadastro
Dropo/jogo fora um banco de dados.

Criação de Banco de Dados com Configuração de Caracteres:

CREATE DATABASE cadastro DEFAULT CHARACTER SET utf8 DEFAULT COLLATE utf8_general_ci
Esse comando define o conjunto de caracteres para `utf8`, permitindo que o banco de dados represente praticamente todos os idiomas do mundo, incluindo acentos, símbolos e caracteres especiais. O `utf8_general_ci` significa que a comparação de strings não será sensível a maiúsculas/minúsculas (case-insensitive) e será feita de forma geral, sem se importar com particularidades de cada idioma.

Comando para Criar a Tabela `pessoas` com uma chave de identificação (Id):

CREATE TABLE pessoas ( id INT NOT NULL AUTO_INCREMENT, nome VARCHAR(30) NOT NULL, nascimento DATE, sexo ENUM('M', 'F'), peso DECIMAL(5,2), -- uso o tipo decimal para definir o peso com precisão de no máximo 5 digitos e 2 digitos após a virgula. altura DECIMAL(3,2), -- aqui eu digo que serão no máximo 3 digitos no total e que após a virgula ira ter dois digitos. nacionalidade VARCHAR(20) DEFAULT 'Brasil', -- caso não seja informada a nacionalidade por padrão será do Brasil. PRIMARY KEY (id) )
Criando uma tabela chamada pessoas, passando as colunas que eu quero ter nela e chave de identificação (não defini o utf8 já que não é regra para criar uma tabela).

- A coluna `id` é do tipo inteiro, não pode ter um valor null/não ter valor definido (NOT NULL) e será gerada automaticamente (AUTO_INCREMENT), garantindo que cada valor seja único.

- A coluna `nome` é do tipo `VARCHAR(30)` e não pode ser nula.

- A coluna `nascimento` é do tipo `DATE`.

- A coluna `sexo` usa o tipo `ENUM`, limitando os valores a 'M' e 'F'.

- As colunas `peso` e `altura` são do tipo `DECIMAL` para garantir precisão em até dois dígitos após a vírgula, nesse formato: (quantidade máximma de digitos, digitos após a vírgula), ou seja se eu definir (5,2) isso significa que eu posso digitar no máx 5 digitos mas 2 terá que ser após a virgula, ex: 110,40.

- A coluna `nacionalidade` tem o valor padrão 'Brasil', caso não seja informada(DEFAULT 'Brasil').

- A chave primária (`PRIMARY KEY`) é definida para a coluna `id`.

Inserir um registro na tabela:

INSERT INTO pessoas (nome, nascimento, sexo, peso, altura) VALUES ('Fulano', '2003-11-13', 'M', 85.50, 1.80);
Os valores passados no parametro será definido ao parametro das colunas da tabela, eu não informei o id porque ele é a primeira coluna da tabela e será gerado automaticamentee, e eu não informei a nascionalidade ela é a última coluna da tabela e por padrão (default) será Brasil caso eu não informe. Obs: O `id` será gerado automaticamente e a `nacionalidade` será 'Brasil' por padrão.

Inserir vários registros de uma só vez:

INSERT INTO pessoas (nome, nascimento, sexo, peso, altura) VALUES ('Fulaninha', '1990-09-21', 'F', 55.00, 1.60), ('Fulanão', '1978-04-27', 'M', 93.00, 1.83), ('Fulanoide', '1992-06-27', 'F', 55.00, 1.55);
Os valores passados nos parametros serão definidoa ao parametro das colunas da tabela, eu não informei o id porque ele é a primeira coluna da tabela e será gerado automaticamente, e a nascionalidade ela é a última coluna da tabela e por padrão (default) será Brasil caso eu não informe. Obs: O `id` será gerado automaticamente e a `nacionalidade` será 'Brasil' por padrão.

Modificar a Coluna `profissao` para Ter 20 Caracteres (Em vez de 10):

ALTER TABLE pessoas MODIFY COLUMN profissao VARCHAR(20)
Altero a tabela pessoas e modifico a coluna profissao mudando a especificação do seu tipo (isso reseta qualquer registro já feito nessa coluna). Obs: eu também poderia mudar o tipo, é como se recriasse a variável.

Definir `profissao` como Não Nula e com Valor Padrão 'Desempregado':

ALTER TABLE pessoas MODIFY COLUMN profissao VARCHAR(20) NOT NULL DEFAULT 'Desempregado'
Alterando a tabela pessoas e modificando a coluna de profissao, mudando as suas especificações do seu tipo. - NOT NULL diz que não pode ser nulo, ou seja cada pessoa tem que ter um valor nessa coluna, caso não tenha será setado por padrão (DEFAULT) 'Desempregado'

Renomear a Coluna `profissao` para `prof`:

ALTER TABLE pessoas CHANGE COLUMN profissao prof VARCHAR(20)
Alterando a tabela pessoas e escolhendo que a coluna 'profissao' se torne 'prof' e sejá um VARCHAR(20), ele recria a variável, logo perde todos os registros feitos anteriormente.

Renomear a Tabela `pessoas` para `infos`:

ALTER TABLE pessoas RENAME TO infos
Altero a tabela pessoas e a renomeio para infos

Criar a Tabela `cursos` se Não Existir

CREATE TABLE IF NOT EXISTS cursos (nome VARCHAR(30) NOT NULL UNIQUE,descricao TEXT,carga INT UNSIGNED,totalaulas INT,ano YEAR DEFAULT '2024') DEFAULT CHARACTER SET utf8
Crio a tabela caso não exista nenhuma outra tabela nesse banco de dados com o nome de cursos, caso exista a tabela não será criada. - 'UNIQUE' nas especificações da coluna 'nome' significa que só podera ter um registro com um nome, ou seja não podera ter mais de um registro com o mesmo nome. - 'UNSIGNED' nas especificações da coluna 'carga' significa que só podera ter valores positivos, ou seja não se pode ter cargas com valores númericos negativos. - 'DEFAULT CHARACTER SET utf8' após os parametros especificando as colunas que a tabela tera, é o mesmo que 'default charset = utf8mb3', só muda a forma de escrever. - A coluna 'ano' é do tipo 'YEAR' e tem o valor padrão de 2024.

Adicionar Chave Primária (Se Esquecida):

ALTER TABLE cursos ADD COLUMN idcurso INT FIRST, ADD PRIMARY KEY (idcurso)
Altero a tabela cursos adiciono uma coluna `idcurso` do tipo INT na primeira posição da tabela como chave primária.

Backup do Banco de Dados (Exportar):

Acesse o menu "Server" no MySQL, depois clique em "Data Export". Selecione os esquemas (bancos de dados) e as tabelas que deseja exportar. Selecione a opção de exportação: - Dump Structure and Data: Exporta tanto a estrutura quanto os dados. - Dump Data Only: Exporta apenas os dados. - Dump Structure Only: Exporta apenas a estrutura, sem dados. Defina o tipo de exportação: - Export to Dump Project Folder: Exporta o projeto completo. - Export to Self-Contained File: Exporta em um único arquivo. Após a configuração, clique em Start Export.

Importação de Banco de Dados (Abrir):

Acesse o menu "Server" e clique em "Data Import". Escolha a opção Import from Dump Project Folder ou Import from Self-Contained File. Selecione o arquivo ou pasta que contém o backup. Clique em Start Import e forneça o usuário e a senha, se necessário.

Selecionar Todos os Registros de uma Tabela:

SELECT * FROM nome_da_tabela
Vai mostrar todos os registros de uma tabela, o todos é representado pelo '*'.

Selecionar Registros Ordenados por Coluna:

SELECT * FROM nome_da_tabela ORDER BY nome
'ORDER BY' é um comando que serve para definir a ordem na qual uma tabela é exibida, no comando mostrado a tabela será exibida com base na coluna nome, ficara em ordem alfabética, se fosse uma coluna de número ficaria em ordem crescente.

Selecionar Registros em Ordem Crescente (Padrão):

SELECT * FROM nome_da_tabela ORDER BY numero ASC
Uso o 'order by' com a especificação da ordem em 'asc' que significa crescente, por padrão o 'order by' sempre será em ordem crescente, portanto é indiferente usar o asc.

Selecionar Registros em Ordem Decrescente:

SELECT * FROM nome_da_tabela ORDER BY numero DESC
Uso o 'order by' com a especificação da ordem em 'desc' que significa decrescente, pór padrão fica em ordem crescente, então se eu quiser que fique em ordem decrescente eu preciso especificar com o 'DESC'.

Selecionar Apenas Determinadas Colunas de uma Tabela:

SELECT nome, carga, ano FROM nome_da_tabela
Invés de selecionar todas colunas de uma tabela com o '*' eu posso passar especificando as colunas que eu quero, no caso desse código só vai exibiar a coluna 'nome', 'carga', 'ano'

Selecionar Registros com Ano Menor ou Igual a 2015:

SELECT * FROM nome_da_tabela WHERE ano <= 2015
Exibo todos os registros de uma tabela cujo o os registros da coluna 'ano' sejam menor ou igual a 2015.

Selecionar Registros com Ano Maior que 2016 em Ordem Decrescente:

select * from nome_da_tabela where ano > 2016 order by ano desc
Exibo todos os registros de uma tabela cujo o os registros da coluna 'ano' sejam maior que 2016, a exibição será em ordem descrescente (ORDER BY ANO DESC).

Selecionar Registros com Ano Diferente de 2017:

SELECT * FROM nome_da_tabela WHERE ano != 2017
Exibo todos os registros de uma tabela cujo o os registros da coluna 'ano' sejam diferente de 2017. Simbolo de diferente: !=

Selecionar Registros com Ano Entre 2014 e 2016:

SELECT * FROM nome_da_tabela WHERE ano BETWEEN 2014 AND 2016
Exibo todos os registros de uma tabela cujo o os registros da coluna 'ano' sejam entre 2014 e 2016. - 'between' significa 'entre' e o 'and' significa 'e'.

Selecionar Registros com Ano de 2014 ou 2017:

SELECT * FROM nome_da_tabela WHERE ano IN (2014, 2017);
Exibo todos os registros de uma tabela cujo o os registros da coluna 'ano' sejam 2014 ou 2017, especifico isso com o IN (valor,valor).

Selecionar Registros com Ano 2014 e Carga 40:

SELECT * FROM nome_da_tabela WHERE ano = 2014 AND carga = 40
Exibo todos os registros de uma tabela cujo o os registros das colunas 'ano' e 'carga' tenham o valor de que o ano seja '2014' e a carga seja '40'. Obs: como eu uso o 'AND' as duas condições tem que ser verdadeiras, o ano precisa ser 2014 e a carga também precisa ser 40. Se eu quisesse que só fosse necessária uma condição ser verdadeira ou as duas serem, eu usaria 'OR' ao invés de 'AND'.

SELECT * FROM nome_da_tabela WHERE ano = 2017 OR carga = 30
Exibo todos os registros de uma tabela cujo o os registros das colunas 'ano' e 'carga' tenham o valor de que o ano seja '2017' ou a carga seja '30'.

O que são WildCards?

Wildcards (ou "caracteres curinga") são símbolos usados em consultas para representar padrões de correspondência em dados. O operador LIKE é utilizado com esses wildcards para realizar buscas parciais em colunas de texto.

Um dos Principais Wildcards é o Percentual (%):

SELECT * FROM clientes WHERE nome LIKE 'Jo%'
A '%'Representa zero ou mais caracteres em uma string. Esse comando retorna todos os nomes que começam com "Jo", como "João", "Jose", "Jonas", etc.

Um dos Principais Wildcards é o Underline (_)

SELECT * FROM clientes WHERE nome LIKE 'J_ão'
o '_' Representa exatamente um único caractere em uma posição específica. Esse comando retorna todos os nomes que começam com "J", seguido de qualquer caractere, e terminam com "ão" (como "João", "Jão", etc).

Diferença entre os Wildcards:

O '_' significa apenas um caractere. A '%' pode representar zero, um ou vários caracteres.
Esses wildcards tornam as buscas no banco de dados mais flexíveis, permitindo a pesquisa por padrões de dados em vez de valores exatos.

Forma ERRADA de filtrar um valor que comece com 'A'(sem wildcard):

SELECT * FROM nome_da_tabela WHERE nome = 'A';
Dessa forma eu vou apenas procurar por por um nome que seja 'A' e não o que comece com a letra 'A'.

Forma CERTA de filtrar um valor que comece com 'A'(com wildcard):

SELECT * FROM nome_da_tabela WHERE nome LIKE 'a%'
Com a implementação do 'LIKE 'a%'' eu condiciono o meu código a fazer a seleção cujo o valor da coluna nome COMECE com a letra 'a'. 'a%' significa que o valor começa com a letra 'a' e após o 'a' pode ter qualquer coisa escrita, o qualquer coisa escrita é representado pela '%'. No caso o a busca pelo A é só um exemplo, poderia ser nada ou qualquer letra ou frase (nenhum caracter, qualquer caracter ou conjunto de caracter).

Filtrar por um valor que termine com a letra 'a':

SELECT * FROM nome_da_tabela WHERE nome LIKE '%a'
Com a implementação do 'LIKE '%a'' eu condiciono o meu código a fazer a seleção cujo o valor da coluna nome TERMINE com a letra 'a'. '%a' significa que pelo final de qualquer coisa escrita ou nada escrito vai ter o 'a'.

Filtrar por um valor que contenha a letra 'a' em qualquer posição:

SELECT * FROM nome_da_tabela WHERE nome LIKE '%a%'
'%a%' significa que o ar pode estár cercado de qualquer caracter ou de nenhum caracter.

Filtrar por um valor que não comece com a letra 'A':

SELECT * FROM nome_da_tabela WHERE nome NOT LIKE 'a%'
O 'NOT' aplicado dessa forma inverte o valor do 'LIKE 'a%''.

Filtrar por um valor que não termine com a letra 'a':

SELECT * FROM nome_da_tabela WHERE nome NOT LIKE '%a'
O 'NOT' aplicado dessa forma inverte o valor do 'LIKE '%a''.

Filtrar por um valor que não contenha a letra 'a':

SELECT * FROM nome_da_tabela WHERE nome NOT LIKE '%a%'
O 'NOT' aplicado dessa forma inverte o valor do 'LIKE '%a%''.

Buscar um valor que comece com 'al' e termine com 'os':

SELECT * FROM nome_da_tabela WHERE nome LIKE 'al%os'
No codigo mostrado o 'al%os' busca por um valor da coluna nome que comece com 'al' e tenha qualquer coisa(%) entre o 'os'. Busca nomes que comecem com 'al' e terminam com 'os'.

Buscar um valor que comece com 'al', tenha qualquer coisa entre 'al' e 'os', e depois tenha algo ou nada após 'os':

SELECT * FROM nome_da_tabela WHERE nome LIKE 'al%os%'

Buscar um valor que tenha um único caractere após 'al' e termine com 'os':

SELECT * FROM nome_da_tabela WHERE nome LIKE 'al%os_'
Combinando a '%' com o '_', a '%' significa nenhum, um ou vários caracteres, já o '_' significa APENAS UM caracter. Isso significa que eu busco por algo que comece com 'al' e termine com algum caracter após o 'os' no final.

DISTINCT:

SELECT DISTINCT ano FROM cursos
Remove valores duplicados e retorna apenas valores únicos. Esse comando retorna apenas os anos distintos da tabela de cursos.

GROUP BY:

SELECT carga, COUNT(nome) FROM cursos GROUP BY carga ORDER BY carga
Agrupa os resultados de uma consulta com base em uma ou mais colunas e frequentemente é usado com funções agregadas como COUNT(), SUM(), AVG(), etc. Esse comando retorna as diferentes cargas horárias de cursos e quantos cursos existem para cada carga horária. 'COUNT(nome)' esse comando conta quantos registros na coluna nome tem.

Função HAVING

SELECT ano, COUNT(*) FROM cursos GROUP BY ano HAVING COUNT(*) > 2
A função HAVING é usada para filtrar os resultados após o agrupamento com GROUP BY. Esse comando retorna os anos que aparecem mais de duas vezes. o 'COUNT(*)' significa que ira contar todos os registros de todas as colunas.

Diferença entre WHERE e HAVING:

WHERE: Filtra os registros antes do agrupamento. HAVING: Filtra os registros após o agrupamento.

Exemplo combinando WHERE e HAVING:

SELECT ano, COUNT(*) FROM cursos WHERE ano > 2012 GROUP BY ano HAVING COUNT(*) > 2
Nesse exemplo, filtramos os anos maiores que 2012, agrupamos para mostrar apenas os anos que se repetem em mais de dois cursos.

Para adicionar uma chave estrangeira em uma tabela, use o comando:

ALTER TABLE nome_da_tabela_que_recebe_fk ADD FOREIGN KEY (nome_da_coluna_que_vai_ser_fk) REFERENCES tabela_que_tem_a_fk (coluna_da_tabela_que_tem_a_fk)
Altero uma tabela adicionando uma foreign key (chave estrangeira), passo no parametro a coluna da tabela que recebera a chave estrangeira e faço a ligação com o 'REFERENCES' assim ligando a tabela que tem a coluna que eu quero que seja a chave estrangeira.

Adicionar um curso preferido para os alunos:

ALTER TABLE alunos ADD COLUMN curso_preferido INT ALTER TABLE alunos ADD FOREIGN KEY (curso_preferido) REFERENCES cursos(id_do_curso)
Altero uma tabela chamada alunos e adiciono a coluna 'curso_preferido' do tipo int. Altero a tabela alunos e adiciono a chave estrangeira que será 'curso_preferido' que recebera o valor de id_do_curso da tabela cursos

Atualizando a chave estrangeira:

UPDATE alunos SET curso_preferido = 3 WHERE id = 5
Atualizando a tabela alunos e setando o curso preferido equivalente ao id_do_curso 3 para o aluno equivalente ao id '5'. Obs: 'curso_preferido' é uma chave estrangeira que referencia a coluna id_do_curso da tabela cursos.

Usando JOIN para exibir dados relacionados:

SELECT alunos.nome, alunos.curso_preferido, cursos.nome FROM alunos JOIN cursos ON cursos.id_do_curso = alunos.curso_preferido
Usando o JOIN, combino as tabelas alunos e cursos para exibir o nome do aluno, o curso preferido e o nome do curso (Uso tanto dados da tabela alunos quanto da tabela cursos). O 'ON' é usado para especificar a condição de junção, neste caso, onde o id_do_curso da tabela cursos é igual ao curso_preferido da tabela alunos.

Usando LEFT JOIN para incluir todos os alunos, mesmo os que não têm curso preferido:

SELECT alunos.nome, alunos.curso_preferido, cursos.nome FROM alunos LEFT JOIN cursos ON cursos.id_do_curso = alunos.curso_preferido
Usando o LEFT JOIN, combino as tabelas alunos e cursos para exibir o nome do aluno, o curso preferido e o nome do curso (Uso tanto dados da tabela alunos quanto da tabela cursos). O 'LEFT JOIN' é usado para incluir todos os alunos, mesmo os que não têm curso preferido (aparecerão os dados da esquerda (LEFT), independente de existir os da direita).

Por que usar LEFT JOIN e não RIGHT JOIN?

O LEFT JOIN garante que todos os registros da tabela à esquerda (alunos) apareçam, mesmo que não tenham correspondência na tabela à direita (cursos). Já o RIGHT JOIN daria prioridade aos cursos, e poderia mostrar registros de cursos sem alunos.

Selecionando o valor Mínimo e o Máximo de um registro:

SELECT MIN(carga_horaria) FROM cursos -- Valor mínimo SELECT MAX(carga_horaria) FROM cursos -- Valor máximo
Seleciono o mínimo e máximo da carga horária dos cursos.

SELECT COUNT() FROM cursos
Contagem total de registros na tabela cursos.

Contando valores específicos:

SELECT COUNT(nome) FROM alunos
Contagem total de nomes na tabela alunos.

Operador BETWEEN:

SELECT nome, ano FROM cursos WHERE ano BETWEEN 2014 AND 2020
O BETWEEN é usado para selecionar registros dentro de um intervalo de valores.
